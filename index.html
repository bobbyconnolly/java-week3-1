Enter ArrayList: Your New Best Friend! 📦

Java provides a wonderful solution called ArrayList. Think of it as a magical box that can:

  • Hold as many pets as you want 📦
  • Grow or shrink as needed 🪄
  • Let you easily add and remove pets 🐾
  • Help you find specific pets quickly 🔍

Let's Create Our First ArrayList!

Create a new package called week3 and add this to your new Main class:

import java.util.ArrayList;

public class Main {
    public static void main(String[] args) {
        // Create an ArrayList to store pets
        ArrayList<Pet> pets = new ArrayList<>();

        // Add some pets
        pets.add(new Dog("Buddy", 3));
        pets.add(new Cat("Whiskers", 5));
        pets.add(new Fish("Nemo", 1, WaterType.SALTWATER));

        // Print how many pets we have
        System.out.println("Total pets: " + pets.size());
    }
}
What's with the <Pet>? This tells Java that our ArrayList will only store Pet objects (and their subclasses like Dog, Cat, and Fish). It's called a "generic type parameter" - we'll learn more about these later! 🎓

Basic ArrayList Operations

Let's learn some essential ArrayList methods:

// Adding pets
pets.add(new Dog("Max", 2));        // Adds at the end
pets.add(0, new Cat("Luna", 1));   // Adds at index 0

// Removing pets
pets.remove(0);                     // Removes pet at index 0
pets.remove(buddy);                  // Removes specific pet object

// Getting pets
Pet firstPet = pets.get(0);        // Gets pet at index 0
int totalPets = pets.size();      // Gets total number of pets

// Checking for pets
boolean hasPet = pets.contains(buddy);  // Checks if pet is in list
boolean isEmpty = pets.isEmpty();       // Checks if list is empty

Different Ways to Loop Through Pets 🔄

Java gives us several ways to loop through an ArrayList. Let's explore them!

1. The For-Each Loop (Recommended! 👍)

// Simple and clean - great for most cases
for (Pet pet : pets) {
    System.out.println(pet.getName() + " says " + pet.makeSound());
}

2. Traditional For Loop

// Good when you need the index number
for (int i = 0; i < pets.size(); i++) {
    Pet pet = pets.get(i);
    System.out.println((i + 1) + ". " + pet.getName());
}

3. Iterator (Advanced Usage)

import java.util.Iterator;

// Useful when you need to remove items while looping
Iterator<Pet> iterator = pets.iterator();
while (iterator.hasNext()) {
    Pet pet = iterator.next();
    if (pet.getAge() > 10) {
        iterator.remove();  // Safely remove old pets
    }
}
Warning: Never remove items from an ArrayList while using a for-each loop! It will cause errors. Use an Iterator instead! 🚫

Common ArrayList Operations 🛠️

Let's create some useful methods for our pet shop:

public static ArrayList<Pet> findPetsByType(ArrayList<Pet> pets, 
                                          Class<?> type) {
    ArrayList<Pet> result = new ArrayList<>();
    
    for (Pet pet : pets) {
        if (type.isInstance(pet)) {  // Check if pet is of given type
            result.add(pet);
        }
    }
    
    return result;
}

public static void adoptPet(ArrayList<Pet> pets, String name) {
    for (int i = 0; i < pets.size(); i++) {
        if (pets.get(i).getName().equals(name)) {
            pets.remove(i);
            System.out.println("🎉 " + name + " has been adopted!");
            return;
        }
    }
    System.out.println("😕 Sorry, couldn't find a pet named " + name);
}
Pro Tip: Notice how findPetsByType returns a new ArrayList? This is a common pattern - create a new list for filtered results! 📝

Let's Use These Methods! 🎯

// Find all dogs in our shop
ArrayList<Pet> dogs = findPetsByType(pets, Dog.class);
System.out.println("We have " + dogs.size() + " dogs!");

// Adopt a pet
adoptPet(pets, "Buddy");  // Bye Buddy! 👋

// Check our updated inventory
displayPets(pets);

Introducing LinkedList: Another Way to Store Pets! 🔄

ArrayList is great, but Java gives us another option called LinkedList. Let's see when we might want to use it! 

import java.util.LinkedList;

public static void main(String[] args) {
    LinkedList<Pet> waitingList = new LinkedList<>();
    
    // Add pets to waiting list
    waitingList.addFirst(new Dog("Max", 2));      // Adds at start
    waitingList.addLast(new Cat("Luna", 1));      // Adds at end
    
    // Remove from either end
    Pet nextPet = waitingList.removeFirst();  // Get next in line
    Pet lastPet = waitingList.removeLast();   // Remove from end
}
Key Differences:
  • ArrayList = like a parking lot (numbered spots) 🅿️
  • LinkedList = like a chain of pets holding hands 🤝
  • ArrayList is better for random access (get(5))
  • LinkedList is better for adding/removing at start/end

Real-World Scenario: Pet Shop Waiting List 🎫

Let's create a waiting list system for our pet shop!

public class PetWaitingList {
    private LinkedList<Pet> waitingList = new LinkedList<>();
    
    public void addToWaitingList(Pet pet) {
        waitingList.addLast(pet);
        System.out.println(pet.getName() + 
            " added to position " + waitingList.size());
    }
    
    public Pet getNextPet() {
        if (waitingList.isEmpty()) {
            System.out.println("No pets in waiting list!");
            return null;
        }
        return waitingList.removeFirst();
    }
    
    public void displayWaitingList() {
        if (waitingList.isEmpty()) {
            System.out.println("Waiting list is empty!");
            return;
        }
        
        System.out.println("🎫 Current Waiting List 🎫");
        int position = 1;
        for (Pet pet : waitingList) {
            System.out.println(position++ + ". " + 
                pet.getName() + " (" + 
                pet.getClass().getSimpleName() + ")");
        }
    }
}

When to Use ArrayList vs LinkedList 🤔

Operation ArrayList LinkedList
Access by index Fast ⚡ Slow 🐌
Add/remove at start Slow 🐌 Fast ⚡
Add/remove at end Fast ⚡ Fast ⚡
Memory usage Less 🎯 More 📦
